## 1. Purpose

### Why This Feature Exists

Turn order visibility is essential for tactical combat planning. Without knowing who acts next, players cannot make informed positioning or ability decisions. The turn order panel transforms combat from reactive guessing to proactive strategy, directly supporting the **Tactical Combat** design pillar. Players need to see the full turn sequence to plan flanking maneuvers, coordinate soldier orders, and time healing abilities effectively.

### What It Enables

- Players can see the complete initiative sequence for all units
- Clear indication of whose turn is currently active
- Ability to plan multiple turns ahead (e.g., "Lyra acts before the archer, so she can Shadowstep behind it")
- Visual feedback when units die (removed from turn order)
- Foundation for turn system (Task 1.8) to manage turn cycling

### Success Criteria

| Criterion | Measurement |
|-----------|-------------|
| Turn order visibility | All units visible in sequence at all times |
| Current turn clarity | Active unit highlighted within 0.1 seconds of turn start |
| Dead unit removal | Removed from panel within 0.5 seconds of death |
| Information density | Show unit type/identity without overwhelming detail |
| Screen real estate | Panel takes minimal space, doesn't obscure grid |

## 3. How It Works

### Overview

The Turn Order Panel is a horizontal bar displayed at the top center of the screen during combat. It shows a sequence of unit icons representing the initiative order from left (current/next) to right (later turns). Each icon displays the unit's identifier (letter or portrait placeholder) and is colored to indicate faction (blue/purple for party, teal for soldiers, red for enemies).

The leftmost icon represents the unit whose turn is currently active and is highlighted with a gold (#f1c40f) border. As turns progress, icons shift left - the current unit's icon animates out when their turn ends, and all remaining icons slide left. When a unit dies, their icon fades out and is removed from the sequence.

The panel uses a semi-transparent dark background (#1a1a2e at 80% opacity) to remain visible against any battlefield without completely obscuring the action. The panel should show up to 12 unit icons (maximum expected combat size: 4 party + 4 soldiers + 4 enemies = 12 units).

### User Flow

```
1. Battle starts - initiative rolled for all units
2. Turn Order Panel populates with all unit icons in initiative sequence
3. Leftmost icon highlighted with gold border (current turn)
4. Player/AI takes action, turn ends
5. Current icon animates out (slides left off panel)
6. All icons shift left smoothly
7. New leftmost icon highlighted
8. If unit dies during combat:
   a. Their icon fades out over 0.3 seconds
   b. Remaining icons shift to fill gap
9. Battle ends - panel hides
```

### Rules & Constraints

| Rule | Description |
|------|-------------|
| Fixed position | Panel always at top center, doesn't move during combat |
| Initiative order | Order determined at battle start, doesn't change mid-battle (except deaths) |
| All units shown | Party, soldiers, AND enemies visible in turn order |
| Faction colors | Blue tones for party, teal for soldiers, red for enemies |
| Current turn highlight | Gold border on active unit's icon |
| Maximum icons | Display up to 12 icons (handles prototype's maximum battle size) |

### Edge Cases

| Scenario | Behavior |
|----------|----------|
| Unit dies on their turn | Icon removed after death animation, turn advances to next |
| All enemies die | Victory triggered, panel fades out with victory screen |
| All party/soldiers die | Defeat triggered, panel fades out with defeat screen |
| Tie initiative rolls | Friendly units act before enemies at same initiative |
| Only 1 unit remaining | Panel still shows, single icon highlighted |
| Unit stunned (future) | Icon shows stun indicator, skipped in turn order |

### Script Responsibilities

**turn_order_panel.gd:**
- Initialize panel when battle starts
- Receive sorted initiative list from TurnManager
- Create TurnOrderIcon instances for each unit
- Handle `turn_advanced` signal to shift icons
- Handle `unit_died` signal to remove icons
- Animate icon transitions (slide, fade)

**turn_order_icon.gd:**
- Store reference to associated Unit
- Display unit's letter and color
- Handle highlight state (current turn)
- Animate fade-out on death

### Detailed Script: turn_order_panel.gd

```gdscript
## TurnOrderPanel - Displays combat turn sequence
## Part of: Blood & Gold Prototype
## Spec: docs/features/1.6-turn-order-ui-panel.md
extends Control

# ===== CONSTANTS =====
const ICON_SIZE: Vector2 = Vector2(40, 40)
const ICON_SPACING: int = 4
const PANEL_PADDING: int = 8
const TRANSITION_DURATION: float = 0.2
const FADE_DURATION: float = 0.3
const COLOR_PANEL_BG: Color = Color("#1a1a2e", 0.8)
const COLOR_CURRENT_TURN: Color = Color("#f1c40f")

# ===== PRELOADS =====
const TurnOrderIconScene = preload("res://scenes/UI/TurnOrderIcon.tscn")

# ===== NODE REFERENCES =====
@onready var panel_bg: Panel = $PanelBackground
@onready var icon_container: HBoxContainer = $PanelBackground/IconContainer

# ===== INTERNAL STATE =====
var _icons: Array[Control] = []
var _current_turn_index: int = 0

# ===== LIFECYCLE =====
func _ready() -> void:
	visible = false  # Hidden until battle starts
	_setup_panel_style()

func _setup_panel_style() -> void:
	var style = StyleBoxFlat.new()
	style.bg_color = COLOR_PANEL_BG
	style.corner_radius_top_left = 6
	style.corner_radius_top_right = 6
	style.corner_radius_bottom_left = 6
	style.corner_radius_bottom_right = 6
	style.content_margin_left = PANEL_PADDING
	style.content_margin_right = PANEL_PADDING
	style.content_margin_top = PANEL_PADDING
	style.content_margin_bottom = PANEL_PADDING
	panel_bg.add_theme_stylebox_override("panel", style)

# ===== PUBLIC API =====
func initialize_turn_order(units: Array[Unit]) -> void:
	## Setup panel with sorted initiative order
	_clear_icons()

	for unit in units:
		var icon = _create_icon(unit)
		_icons.append(icon)
		icon_container.add_child(icon)

	_current_turn_index = 0
	_update_current_turn_highlight()
	visible = true
	print("[TurnOrderPanel] Initialized with %d units" % units.size())

func advance_turn() -> void:
	## Move to next unit in turn order
	if _icons.is_empty():
		return

	# Remove current icon with animation
	var current_icon = _icons[0]
	_animate_icon_out(current_icon)
	_icons.remove_at(0)

	# If more turns remain, highlight new current
	if not _icons.is_empty():
		_update_current_turn_highlight()

func remove_unit(unit: Unit) -> void:
	## Remove a unit from turn order (death)
	for i in range(_icons.size()):
		if _icons[i].get_unit() == unit:
			var icon = _icons[i]
			_icons.remove_at(i)
			_animate_icon_death(icon)
			break

	# Update highlight if needed
	if not _icons.is_empty():
		_update_current_turn_highlight()

func hide_panel() -> void:
	## Hide panel at end of battle
	visible = false
	_clear_icons()

# ===== INTERNAL METHODS =====
func _create_icon(unit: Unit) -> Control:
	var icon = TurnOrderIconScene.instantiate()
	icon.setup(unit)
	icon.custom_minimum_size = ICON_SIZE
	return icon

func _clear_icons() -> void:
	for icon in _icons:
		icon.queue_free()
	_icons.clear()

func _update_current_turn_highlight() -> void:
	for i in range(_icons.size()):
		_icons[i].set_current_turn(i == 0)

func _animate_icon_out(icon: Control) -> void:
	var tween = create_tween()
	tween.tween_property(icon, "modulate:a", 0.0, TRANSITION_DURATION)
	tween.tween_callback(icon.queue_free)

func _animate_icon_death(icon: Control) -> void:
	var tween = create_tween()
	tween.tween_property(icon, "modulate:a", 0.0, FADE_DURATION)
	tween.tween_callback(icon.queue_free)
```

### Detailed Script: turn_order_icon.gd

```gdscript
## TurnOrderIcon - Individual unit icon in turn order panel
## Part of: Blood & Gold Prototype
extends Control

# ===== CONSTANTS =====
const BORDER_WIDTH: int = 3
const COLOR_CURRENT_TURN: Color = Color("#f1c40f")

# ===== NODE REFERENCES =====
@onready var background: Panel = $Background
@onready var letter_label: Label = $LetterLabel
@onready var current_border: Panel = $CurrentBorder

# ===== STATE =====
var _unit: Unit = null
var _is_current_turn: bool = false

# ===== PUBLIC API =====
func setup(unit: Unit) -> void:
	_unit = unit
	_update_appearance()

func get_unit() -> Unit:
	return _unit

func set_current_turn(is_current: bool) -> void:
	_is_current_turn = is_current
	current_border.visible = is_current

# ===== INTERNAL =====
func _update_appearance() -> void:
	if not _unit:
		return

	# Set letter
	letter_label.text = Unit.UNIT_LETTERS.get(_unit.unit_type, "?")

	# Set background color
	var bg_style = StyleBoxFlat.new()
	bg_style.bg_color = Unit.UNIT_COLORS.get(_unit.unit_type, Color.GRAY)
	bg_style.corner_radius_top_left = 4
	bg_style.corner_radius_top_right = 4
	bg_style.corner_radius_bottom_left = 4
	bg_style.corner_radius_bottom_right = 4
	background.add_theme_stylebox_override("panel", bg_style)
```

## 10. Implementation Notes

### Approach Rationale

**Why HBoxContainer for icons?**
- Automatic spacing and arrangement
- Easy to add/remove children dynamically
- Built-in support for minimum sizes
- Handles overflow gracefully

**Why separate TurnOrderIcon scene?**
- Clean separation of icon logic
- Easy to enhance later (portraits, tooltips)
- Can be instanced/freed without complex management
- Follows Godot's composition pattern

**Why Control node for panel root?**
- Full control over positioning via anchors
- Works well with Godot's UI system
- Easy to add to CanvasLayer for UI layering
- Supports themes and styles

### Gotchas to Watch

1. **Anchor setup**: Panel must be anchored to top-center, not affected by viewport resizing
2. **Z-index/layer**: Panel should be on UI CanvasLayer to render above game elements
3. **Unit reference**: Icons hold Unit references - must handle null if unit freed before icon
4. **Performance**: Don't create/free icons every frame - only on turn changes
5. **Signal timing**: `turn_advanced` must fire before panel updates, not after

### Alternative Approaches Considered

| Approach | Pros | Cons | Decision |
|----------|------|------|----------|
| Vertical panel (side) | More room | Obscures grid edge, unconventional | Rejected |
| Circular/radial layout | Unique, compact | Complex implementation, unusual UX | Rejected |
| Bottom of screen | Near abilities | Competes with ability bar space | Rejected |
| **Horizontal top bar** | Standard, clear | Takes top space | **Chosen** |
| Portrait images | More visual interest | Requires art assets, more complex | Future enhancement |

### Future Enhancements (Out of Scope)

- Unit portraits instead of letters
- Hover tooltip showing unit name and stats
- Preview of next round's turn order
- Estimated damage indicator on hover
- Turn order modification abilities (haste/slow)
